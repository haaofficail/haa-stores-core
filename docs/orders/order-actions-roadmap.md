# Order Actions Roadmap

## Status Machine

```
بدء الطلب → في انتظار الدفع → مؤكد → قيد التجهيز
                                        ↓
                    ┌──────────────────────────────┐
                    ↓                              ↓
            (توصيل)                          (استلام فرع)
                    ↓                              ↓
            جاهز للشحن                      جاهز للاستلام
                    ↓                              ↓
              تم الشحن                         تم الاستلام (نهائي)
                    ↓
            تم التوصيل
                    ↓
               مكتمل (نهائي)

            حالات نهائية أخرى:
            ملغي | مسترد | مسترد جزئياً | مرتجع للمرسل
```

## 4 محاور مستقلة

| المحور | يظهر لـ | نوع |
|--------|---------|-----|
| الشحن والبوليصة | توصيل فقط | كرت داخل الطلب |
| الاستلام من الفرع | استلام فرع فقط | كرت داخل الطلب |
| الهدية | sendAsGift=true أو giftMessage | كرت داخل الطلب |
| الدفع عند الاستلام | cash_on_delivery | كرت الدفع |

لا أحد من هذه المحاور الأربعة يكون "حالة طلب". كلها بيانات وإجراءات داخل الطلب.

## الإجراءات حسب الحالة

| الحالة | الإجراءات الأساسية |
|--------|-------------------|
| في انتظار الدفع | تأكيد الدفع / إلغاء |
| مؤكد | بدء التجهيز / إلغاء |
| قيد التجهيز | جاهز للشحن / جاهز للاستلام / إلغاء |
| جاهز للشحن | إنشاء بوليصة / طباعة بوليصة / تسليم لشركة الشحن / إلغاء |
| تم الشحن | إعادة إرسال بيانات الشحن / فتح التتبع / توصيل / إرجاع |
| تم التوصيل | مكتمل / إرجاع |
| جاهز للاستلام | تأكيد الاستلام / إلغاء |
| تم الاستلام | لا يوجد إجراء تشغيلي (نهائي) |
| مكتمل | لا يوجد إجراء تشغيلي (نهائي) |
| ملغي | لا يوجد إجراء (نهائي) |
| مسترد | لا يوجد إجراء (نهائي) |
| مسترد جزئياً | عرض تفاصيل الاسترداد فقط |
| مرتجع للمرسل | استرجاع / استرداد |

## مبادئ البوليصة

- إنشاء البوليصة لا يعني تم الشحن
- طباعة البوليصة لا تعني تم الشحن
- تسليم شركة الشحن = تم الشحن
- بعد تم الشحن: يعرض رقم التتبع + رابط التتبع فقط (لا يعرض إنشاء بوليصة)

## مبادئ الدفع عند الاستلام (COD)

- في التوصيل: تسليم لشركة الشحن → تم الشحن (paymentStatus يبقى unpaid)
- في التوصيل: تم التوصيل وتحصيل المبلغ → تم التوصيل + مدفوع
- في الاستلام: زر "تأكيد الاستلام" يتحول إلى "تأكيد الاستلام وتحصيل المبلغ"
- عند الضغط: تم الاستلام + مدفوع

## مبادئ الهدية

- الهدية ليست حالة طلب
- تظهر فقط إذا: sendAsGift=true أو giftMessage موجودة أو المستلم مختلف عن المشتري
- الإجراءات: عرض رسالة / طباعة رسالة / إشعار المشتري / إشعار المستلم

## ترتيب صفحة تفاصيل الطلب

1. ملخص الطلب
2. المنتجات
3. الدفع
4. التجهيز
5. التغليف
6. الهدية — فقط إذا الطلب هدية
7. الشحن والتوصيل — فقط إذا الطلب توصيل
8. الاستلام من الفرع — فقط إذا الطلب استلام
9. المشتري والمستلم
10. الفاتورة والمستندات
11. السجل والملاحظات
12. الإجراءات الخطرة

## الخطة التنفيذية

| Phase | الوصف |
|-------|-------|
| 0 | توثيق (هذا الملف) |
| 1 | `getOrderActions(order)` resolver |
| 2 | ترتيب صفحة الطلب حسب الـ resolver |
| 3 | الشحن والبوليصة (إنشاء، طباعة، تسليم) |
| 4 | الاستلام من الفرع (جاهز، تأكيد، تحصيل) |
| 5 | الدفع عند الاستلام (تحصيل، فشل) |
| 6 | الهدية البسيطة (عرض، طباعة، إشعار) |
