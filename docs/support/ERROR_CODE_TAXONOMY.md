# Error Code Taxonomy

> Hierarchical classification of error codes used across the system.
>
> ## Identifier Explanation
>
> Every error event carries three identifiers:
> - **errorCode** (`THEME-001`): Classifies the error type. Used for catalog lookup and severity assignment.
> - **correlationId** (`req_xxxx`): Ties together all events in a single request flow. Generated by the request-id middleware or ErrorBoundary.
> - **eventId** (`evt-xxxx`): Unique identifier for a single error event. Used for deduplication and detailed log lookup.
> - **fingerprint** (`errorCode::area::route::message`): Groups identical errors across requests for dedup and RCA.

## Severity Matrix

| Severity | Definition | Response Time | Example |
|----------|-----------|---------------|---------|
| P0 | Complete service outage | Immediate | Storefront down, Dashboard down |
| P1 | Major feature broken | < 4 hours | Payment failing, theme broken |
| P2 | Feature degraded, workaround exists | < 24 hours | Order state error, permission denied |
| P3 | Minor issue, cosmetic | < 1 week | Validation error, timeout |
| P4 | Backlog / technical debt | Backlog | Missing documentation |

## Source Taxonomy

| Source | Meaning | Examples |
|--------|---------|----------|
| `platform_bug` | Code defect in the platform | Null pointer, wrong state |
| `merchant_config` | Merchant-configured setting is wrong | Invalid webhook URL, wrong API key |
| `external_provider` | Third-party service error | Payment gateway timeout, carrier 500 |
| `permission_denied` | User lacks required role/scope | RBAC check failed |
| `validation_error` | Request payload failed validation | Missing field, wrong type |
| `theme_runtime` | Theme component threw at runtime | Liquid render error, JS exception |
| `network_error` | Network-level failure | DNS failure, connection refused |
| `database_error` | Database-level failure | Connection pool exhausted, deadlock |
| `unknown` | Unclassified | Default for unhandled errors |
