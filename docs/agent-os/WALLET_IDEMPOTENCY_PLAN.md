# Wallet Idempotency Migration Plan — DECISION-OS-018

> **Status:** PLAN ONLY. Migration NOT executed. No `db:migrate` run.
> **Approval gate:** owner must explicitly authorise before the migration is generated and run against any database.

---

## The gap (per QA-C F-QA-C-001)

`packages/db/src/schema/wallet.ts` declares indexes on `walletEntries` but **no unique constraint** for any entry type except `platform_fee` (which has a partial unique index via migration 0062 — see `packages/wallet-core/src/ledger.ts:157-178`).

The other 7 entry types in `WalletPostingService`'s `DedupKey` union (`packages/commerce-core/src/wallet-posting-service.ts:43-48`) rely on an in-memory `dedupMap` that is per-service-instance only. Two parallel webhooks (provider retry, concurrent processing) can each create a fresh `WalletPostingService`, miss the dedup, and write a second wallet entry → double-credit / double-debit.

Affected types:

| Entry type              | Reference  | Direction | Risk if duplicated           |
| ----------------------- | ---------- | --------- | ---------------------------- |
| `sale`                  | order      | credit    | over-credit merchant balance |
| `cod_fee`               | order      | debit     | over-debit                   |
| `refund`                | refund     | debit     | over-debit                   |
| `gateway_fee`           | order      | debit     | over-debit                   |
| `payout_debit`          | payout     | debit     | double payout                |
| `payout_reversal`       | payout     | credit    | over-credit                  |
| `settlement_difference` | adjustment | either    | imbalance                    |

`platform_fee` (already DB-protected) is the template.

---

## Proposed migration

A single new migration file under `packages/db/src/migrations/` (next free number) that adds 7 partial unique indexes mirroring the existing `platform_fee` index:

```sql
-- Auto-generated by drizzle-kit after schema update (do NOT hand-craft).
CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_sale_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'sale' AND "reference_type" = 'order';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_cod_fee_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'cod_fee' AND "reference_type" = 'order';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_refund_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'refund' AND "reference_type" = 'refund';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_gateway_fee_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'gateway_fee' AND "reference_type" = 'order';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_payout_debit_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'payout_debit' AND "reference_type" = 'payout';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_payout_reversal_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'payout_reversal' AND "reference_type" = 'payout';

CREATE UNIQUE INDEX IF NOT EXISTS "wallet_entries_settlement_diff_uniq"
  ON "wallet_entries" ("store_id", "reference_id")
  WHERE "type" = 'settlement_difference' AND "reference_type" = 'adjustment';
```

> **Generation rule:** these indexes must be created by `drizzle-kit generate` after a schema update, not hand-edited. Manual SQL is shown here only for clarity.
