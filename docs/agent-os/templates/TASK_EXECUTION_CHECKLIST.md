# Task Execution Checklist

> **Use:** copy this checklist at the start of a scoped task, then mark items as you complete them.
> **Purpose:** turn a short request into a controlled task with clear scope, verification, and handoff.

---

## 1. Request definition

- [ ] Original user request captured verbatim
- [ ] One-sentence task interpretation written
- [ ] Task type classified

## 2. Scope control

- [ ] In-scope items listed
- [ ] Out-of-scope items listed
- [ ] Affected files or areas identified

## 3. Delivery contract

- [ ] Acceptance criteria written and testable
- [ ] Risks identified
- [ ] Verification plan written

## 4. Execution

- [ ] Existing implementation/docs inspected before changing anything
- [ ] Changes kept inside the agreed scope only
- [ ] Final diff reviewed before completion
- [ ] Relevant checks run

## 5. Documentation sync

- [ ] `docs/ops/TASK_TRACKER.md` updated
- [ ] `docs/ops/CURRENT_STATE.md` updated when state changed
- [ ] `docs/ops/ISSUE_KNOWLEDGE_BASE.md` updated if a root cause was found
- [ ] `docs/ops/CHANGELOG_INTERNAL.md` updated if the change altered structure, behavior, or process
- [ ] `docs/ops/DECISIONS.md` updated if a decision was made

## 6. Final report

- [ ] What changed?
- [ ] Why did it change?
- [ ] Which files changed?
- [ ] What tests/checks were run?
- [ ] What was not changed?
- [ ] What risks remain?
- [ ] What is the next suggested step?
