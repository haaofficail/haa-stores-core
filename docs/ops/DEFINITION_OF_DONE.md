# Definition of Done

> A task is NOT complete until ALL of these criteria are satisfied.

---

## Completion Criteria

- [ ] **Implementation complete** — required changes are done within scope
- [ ] **No unrelated files changed** — only files identified in the plan were modified
- [ ] **No TypeScript errors** — `pnpm typecheck` passes
- [ ] **Tests pass** — relevant tests are run, OR reason for not running is documented
- [ ] **RTL checked** — UI changes verified on RTL layout
- [ ] **Mobile checked** — UI changes verified on mobile viewport
- [ ] **Permissions enforced in API** — RBAC changes are enforced server-side, not just UI
- [ ] **Theme isolated** — theme changes do not leak to merchant dashboard
- [ ] **TASK_TRACKER updated** — task status and notes recorded
- [ ] **CURRENT_STATE updated** — if project state changed
- [ ] **ISSUE_KNOWLEDGE_BASE updated** — if root cause was identified and fixed
- [ ] **REGRESSION_CHECKLIST updated** — if the issue is regressable
- [ ] **CHANGELOG_INTERNAL updated** — if structural, behavioral, or operational change
- [ ] **DECISIONS updated** — if architectural or process decision was made
- [ ] **Final report written** — summary of what changed, why, and what's next
